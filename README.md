# Weather Prediction ML Project

Machine learning project for predicting rainfall in Australia using MLflow for experiment tracking and DVC for data version control and pipeline management.

## Project Structure

```
mlops_lab_1/
├── .dvc/                   # DVC configuration
├── data/
│   ├── raw/

│   │   └── weatherAUS.csv.dvc   # DVC tracking file
│   └── processed/          # Generated by pipeline (gitignored)
├── models/                 # Generated by pipeline (gitignored)
├── notebooks/
│   └── 01_eda_weatherAUS.ipynb

│   └── train.py           # Model training stage
```

## Setup

### 1. Create Virtual Environment

```bash
python -m venv venv
```

### 2. Activate Virtual Environment

Windows:
```bash
venv\Scripts\activate
```

Linux/Mac:
```bash
source venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Initialize DVC (if not already done)

```bash
dvc init
```

### 5. Pull Data from DVC Remote (if configured)

```bash
dvc pull
```

## Usage

### Run Exploratory Data Analysis

```bash
jupyter notebook notebooks/01_eda_weatherAUS.ipynb
```

### Run DVC Pipeline

The project uses DVC to manage a two-stage ML pipeline:

**Run the entire pipeline:**
```bash
dvc repro
```

This will execute:
1. **Prepare stage**: Load raw data, preprocess, split into train/test sets
2. **Train stage**: Train model, evaluate, save artifacts

**View metrics:**
```bash
dvc metrics show
```

**View pipeline DAG:**
```bash
dvc dag
```

### Modify Pipeline Parameters

Edit `params.yaml` to change pipeline parameters:

```yaml
prepare:
  test_size: 0.2
  random_state: 42

train:
  model_type: random_forest
  n_estimators: 100
  max_depth: 15
  min_samples_split: 2
```

Then run `dvc repro` to execute only the affected stages.

### DVC Caching Demonstration

**Test 1 - No changes (full cache hit):**
```bash
dvc repro
# Output: "Data and pipelines are up to date."
```

**Test 2 - Change training parameters:**
```bash
# Edit params.yaml: change n_estimators to 200
dvc repro
# Output: Only 'train' stage runs, 'prepare' is skipped
```

**Test 3 - Change data preparation parameters:**
```bash
# Edit params.yaml: change test_size to 0.3
dvc repro
# Output: Both 'prepare' and 'train' stages run
```

**Test 4 - Modify source code:**
```bash
# Edit src/train.py
dvc repro
# Output: Only 'train' stage runs
```

### Push/Pull Data with DVC

**Pull from remote storage:**

## Dataset

Weather Australia dataset containing:
- 145,460 observations
- 23 features (weather measurements)
- Target: RainTomorrow (Yes/No)

Features include temperature, humidity, pressure, wind direction, rainfall, and more.

The raw dataset is tracked with DVC (`data/raw/weatherAUS.csv.dvc`).

## Pipeline Stages

### Stage 1: Prepare
- **Input**: Raw data (`data/raw/weatherAUS.csv`)
- **Output**: Processed data (`data/processed/`)
- **Parameters**: `test_size`, `random_state`
- **Tasks**:
  - Load raw data
  - Handle missing values
  - Encode categorical variables
  - Scale numerical features
  - Split into train/test sets

### Stage 2: Train
- **Input**: Processed data (`data/processed/`)
- **Output**: Model (`model.pkl`), Metrics (`metrics.json`)
- **Parameters**: `model_type`, `n_estimators`, `max_depth`, `min_samples_split`
- **Tasks**:
  - Load preprocessed data
  - Train ML model
  - Evaluate on test set
  - Save model and metrics
  - Log to MLflow

## Key Benefits of DVC Integration

1. **Reproducibility**: Pipeline stages are fully reproducible with `dvc repro`
2. **Caching**: DVC only re-runs stages when dependencies change
3. **Data Versioning**: Large datasets tracked efficiently without bloating git
4. **Experiment Tracking**: Combined with MLflow for comprehensive tracking
5. **Collaboration**: Share data and models through remote storage
6. **Parameter Management**: Centralized configuration in `params.yaml`
